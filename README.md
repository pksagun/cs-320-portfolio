Throughout this course, I learned that writing functional and secure software requires careful planning, testing, and validation. First, I made sure that each class followed the project requirements
by checking inputs and preventing invalid data from being stored. I also created unit tests using JUnit to verify that each method worked as expected and to catch errors before they became bigger problems. 
Additionally, testing different situations, including invalid inputs and boundary values, helped me build more reliable software.

I learned that understanding user needs is an important part of software development. Before writing code, I carefully reviewed the project requirements to understand what the user expected the program to do. 
Then, I designed my classes and methods to meet those requirements while keeping the code organized and easy to maintain. Finally, I approached software design by breaking larger problems into smaller parts and giving 
each class a specific responsibility. This made the code easier to read, test, and update. Overall, this course strengthened my understanding of software testing, object-oriented programming, and the importance of creating software 
that is both reliable and maintainable.
